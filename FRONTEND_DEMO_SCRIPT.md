# Frontend Prototype Demo Script

**Duration:** 3 minutes  
**Purpose:** Show working UI prototypes to demonstrate design implementation

---

## üé¨ Demo Script

### Opening Statement
*"In addition to our architecture and documentation, I've also started implementing our UI designs. Let me show you four working prototypes."*

---

### Page 1: Home Page (30 seconds)

**Open:** `home.html` in browser

**Say:**
*"This is our landing page with a gradient hero section using our primary colors - pink for PCOS awareness and blue for healthcare trust. Notice the three feature cards for our modules: PCOS-Detection, Moodie, and Foodie."*

**Show:**
- Scroll to features section
- Hover over cards (they lift on hover)
- Show stats section
- Show responsive navigation

---

### Page 2: Signup Page (30 seconds)

**Click:** "Get Started" button OR navigate to `signup.html`

**Say:**
*"Our registration form includes real-time validation and a password strength indicator. Watch what happens as I type a password."*

**Show:**
- Type in password field
- Show strength bar changing colors
- Point out form validation
- Show terms & conditions checkbox
- Mention social login placeholders for future

---

### Page 3: Login Page (30 seconds)

**Click:** "Login here" link OR navigate to `login.html`

**Say:**
*"The login page features password visibility toggle, remember me option, and forgot password functionality - all standard security features we'll implement in the backend phase."*

**Show:**
- Click eye icon to toggle password visibility
- Show remember me checkbox
- Point out demo credentials note
- Click "Dashboard" quick access button

---

### Page 4: Dashboard (60 seconds)

**Navigate to:** `dashboard.html`

**Say:**
*"This is the main user dashboard with a fixed sidebar for easy navigation. Users can see their stats at a glance and access all three modules from here."*

**Show:**
- **Sidebar:** Point out navigation items
  - Dashboard, PCOS Prediction, Moodie, Foodie, Reports, Profile
- **Stats Bar:** Show 0 entries (new user)
- **Module Cards:** Hover over each to show interaction
  - PCOS-Detection (pink)
  - Moodie Journal (blue)
  - Foodie Chatbot (green)
- **Click a card:** Show "coming soon" alert
  - *"This demonstrates these will be implemented in later phases"*

---

### Responsive Design Demo (20 seconds)

**Say:**
*"All pages are mobile-responsive using Bootstrap 5. Let me show you..."*

**Show:**
- Resize browser window
- Show how cards stack on mobile
- Show collapsible navbar

---

### Closing Statement (10 seconds)

**Say:**
*"These prototypes demonstrate our UI/UX design implementation and will be converted to EJS templates when we integrate with the backend in Week 9-10. The design follows our principles of empathetic, accessible, and mobile-first development."*

## üí° If Professor Asks...

### "Why static HTML instead of integrated with backend?"
**Answer:**  
*"Following agile methodology, we're doing frontend and backend development in parallel. These prototypes help us validate the UI/UX before integration. Backend API development is scheduled for Week 7-8, then we'll integrate in Week 9-10."*

### "What about the other pages (journal, chatbot, prediction)?"
**Answer:**  
*"We've prioritized the core user flow for this checkpoint. The other pages will follow the same design pattern. We have the wireframes and guidelines documented, so implementation will be consistent."*

### "Is this mobile-responsive?"
**Answer:**  
*"Yes, all pages use Bootstrap 5's responsive grid system. Let me demonstrate..."* [Resize browser]

### "What about browser compatibility?"
**Answer:**  
*"We're using modern web standards (HTML5, CSS3, ES6) and Bootstrap 5, which supports all major browsers. We'll test on Chrome, Firefox, Safari, and Edge during the testing phase (Week 13-14)."*

### "Can users actually sign up?"
**Answer:**  
*"Not yet - these are UI prototypes. The forms have client-side validation, but backend authentication with MongoDB and JWT will be implemented in Week 7-8. Right now, clicking submit shows an alert explaining this is a prototype."*

## üöÄ Demo Tips

1. **Have pages pre-loaded** in browser tabs before the meeting
2. **Practice the flow** 2-3 times beforehand
3. **Keep it smooth** - don't fumble with file navigation
4. **Time it** - stick to 3 minutes
5. **Be confident** - you built working code!
6. **Interact** - click buttons, hover, type in forms
7. **Show responsive** - resize browser to demonstrate
